# Services

Service skills live here. One folder per service, each with a `SKILL.md` following the four canonical sections (Context → Connection → Procedure → Verification & error handling). See the template at `.cursor/skills/_templates/service-skill.md`.

## Conventions

- Folder slug = service slug, `kebab-case` (e.g. `stripe`, `resend`, `vercel`, `supabase`).
- Service skills are **cross-domain** — they may be referenced from any domain's `_domain/SKILL.md`.
- Connection details (env vars, scopes) live in the `Connection` section of the SKILL — not in code or KB.
